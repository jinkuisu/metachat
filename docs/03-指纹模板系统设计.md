# MetaChat 指纹模板系统设计

## CloakBrowser 集成方式

cloakbrowser npm 包提供的是一个带 C++ 级反检测补丁的 Chromium 二进制。
我们的 Rust 后端通过以下方式使用它：

1. npm install cloakbrowser → 下载 Chromium 二进制到 node_modules
2. Rust 通过 Node.js 的 resolve 找到二进制路径
3. process.rs 启动二进制 + --remote-debugging-port
4. 通过 CDP 应用指纹配置
5. 内置的反检测补丁自动生效（不需要注入 JS）

## 指纹模板系统架构

┌─────────────────────────────────────────────────────────────┐
│  Layer 0: Template Presets                                   │
│  内置模板（不可修改）+ 用户自定义模板                          │
│  windows_chrome_120 / windows_chrome_124 / mac_chrome_120    │
├─────────────────────────────────────────────────────────────┤
│  Layer 1: Generator Engine                                   │
│  seed → 确定性生成各维度参数                                  │
│  同一个 seed 永远生成同一套指纹                                │
├─────────────────────────────────────────────────────────────┤
│  Layer 2: CDP Applicator                                     │
│  通过 CDP 命令将指纹写入 CloakBrowser 实例                    │
│  分 4 层：启动参数 / 网络层 / Emulation层 / JS注入层          │
├─────────────────────────────────────────────────────────────┤
│  Layer 3: Storage                                            │
│  SQLite fingerprint_templates 表                             │
│  账号创建时关联一个模板 + seed                                │
├─────────────────────────────────────────────────────────────┤
│  Layer 4: UI                                                 │
│  模板列表 / 编辑器 / 预览 / 账号设置                          │
└─────────────────────────────────────────────────────────────┘

## FingerprintConfig 完整字段

pub struct FingerprintConfig {
    // ── 元数据 ──
    id: String,
    name: String,
    description: String,
    seed: String,           // 生成种子，账号创建时分配
    template_id: String,    // 关联的模板
    is_system: bool,

    // ── Web 客户端身份 (Layer 1: 启动参数) ──
    user_agent: String,
    platform: String,         // Win32 | MacIntel | Linux x86_64
    browser_version: String,  // 120.0.0.0
    hardware_concurrency: u32,
    device_memory_gb: u32,

    // ── 时间 & 区域 (Layer 2: Emulation) ──
    timezone: String,         // Asia/Shanghai
    locale: String,           // zh-CN
    languages: Vec<String>,   // [zh-CN, zh, en]
    accept_language: String,  // zh-CN,zh;q=0.9,en;q=0.8

    // ── 屏幕 (Layer 2: Emulation) ──
    screen_width: u32,
    screen_height: u32,
    color_depth: u32,         // 24 | 30 | 48
    pixel_ratio: f64,         // 1 | 1.25 | 1.5 | 2 | 2.5 | 3
    avail_width: u32,
    avail_height: u32,

    // ── GPU (Layer 3: Emulation) ──
    gpu_vendor: String,
    gpu_renderer: String,

    // ── 指纹噪声 (Layer 3: JS Injection) ──
    canvas_noise: f64,        // 0.001 ~ 0.1
    webgl_noise: f64,         // 0.001 ~ 0.05
    audio_noise: f64,         // 0.0001 ~ 0.01
    client_rects_noise: f64,  // 0.001 ~ 0.05

    // ── 字体 (Layer 3: JS Injection) ──
    fonts: Vec<String>,

    // ── 代理 ──
    proxy: Option<ProxyConfig>,

    created_at: i64,
    updated_at: i64,
}

## 模板预设

每个内置模板定义一套基准参数，generate() 基于 seed 做微小偏移：
- windows_chrome_120  (Windows 10 + Chrome 120)
- windows_chrome_124  (Windows 11 + Chrome 124)
- mac_chrome_120      (macOS 14 + Chrome 120)
- mac_chrome_124      (macOS 15 + Chrome 124)
- linux_chrome_120    (Ubuntu 22.04 + Chrome 120)

用户可以在内置模板基础上克隆修改。

## 生成逻辑

generate(template, seed) -> FingerprintConfig:
  1. SHA256(seed) → 32 字节哈希
  2. 前 4 字节 → i32 种子值
  3. 种子值代入每个维度的确定性函数：
     - 屏幕分辨率：从模板预设的 3-5 个候选中按种子选一个
     - 时区：种子决定偏移量
     - Canvas 噪声：seed % 分布值
     - 字体列表：从候选集中随机选 N 个
  4. UA 中的版本号：模板固定，不随机

## CDP 应用流程

apply_via_cdp(fingerprint, cdp_client):
  Step 1: Network.setUserAgentOverride
           { userAgent, acceptLanguage, platform }

  Step 2: Emulation.setUserAgentOverride
           { userAgent, acceptLanguage, platform }

  Step 3: Emulation.setTimezoneOverride
           { timezoneId }

  Step 4: Emulation.setLocaleOverride
           { locale }

  Step 5: Emulation.setDeviceMetricsOverride
           { width, height, deviceScaleFactor, mobile: false }

  Step 6: Page.addScriptToEvaluateOnNewDocument
           注入 JS: navigator 属性重写 + Canvas/WebGL/Audio 噪声

  Step 7: (如果 CloakBrowser 支持额外 API) 自定义调用

## 存储

SQLite 表 fingerprint_templates:
  id TEXT PK
  name TEXT
  description TEXT
  platform TEXT
  browser TEXT
  is_system INTEGER
  config_json TEXT (FingerprintConfig 的 JSON 序列化)
  created_at INTEGER
  updated_at INTEGER

账号关联：accounts 表加一列 fingerprint_template_id

## UI 规划

1. 模板列表页：内置模板 + 用户模板，支持搜索/过滤
2. 模板编辑器：按维度分组，滑块/下拉/输入
3. 指纹预览：生成后展示当前指纹摘要
4. 账号设置：选择模板 + 生成新指纹
