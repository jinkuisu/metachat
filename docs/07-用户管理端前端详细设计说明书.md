# MetaChat 重构项目 · 用户管理端前端详细设计说明书

## 1. 技术栈

Vue 3 + Vite + Naive UI + Pinia + vue-router + axios
独立 SPA，与运维端路由共用但视图不同

## 2. 双用户视角架构

同一前端 SPA，登录后根据 userType + role 决定渲染哪个视角:

| 用户类型 | 角色 | 视角 |
|----------|------|------|
| sys_user | sales | 销售视角(全功能) |
| mc_user  | customer | 客户自助视角(仅看自己) |

## 3. 销售视角路由

/login                                 - 登录(统一)
/sales/dashboard                      - 销售首页
/sales/clients                        - 客户列表
/sales/clients/create                 - 新增客户
/sales/clients/:id                    - 客户详情
/sales/clients/:id/subscriptions      - 客户订阅
/sales/clients/:id/moderation         - 客户敏感词
/sales/depts                          - 部门管理
/sales/depts/:id/pricing              - 部门定价
/sales/plans                          - 套餐自定义
/sales/fans                           - 粉丝列表
/sales/fans/tasks                     - 接粉任务
/sales/fans/links                     - 活链管理
/sales/moderation/violations          - 违规记录
/sales/backups                        - 备份列表
/sales/backups/search                 - 消息检索
/sales/stats/team                     - 团队统计
/sales/stats/users                    - 人员统计

## 5. 布局结构

LoginPage (共用)
SalesLayout (销售视角)
  +-- Sidebar (客户/部门/粉丝/审核/备份/统计)
  +-- Header (客户搜索 + 通知)
  +-- Content (RouterView)

CustomerLayout (客户视角)
  +-- Sidebar (精简菜单)
  +-- Header (个人)
  +-- Content (RouterView)

## 6. 数据隔离

- 销售只能看到自己部门下(deptId)的客户数据
- 客户只能看到自己(userId)的数据
- 后端 API 通过 JWT 中的 deptId/userId 做数据过滤
- 前端路由守卫: 无权限跳转 403 页面

## 7. 客户详情页设计

/sales/clients/:id 页面:
  NCard 基本信息(用户名, 手机, 邮箱, 状态, 套餐)
  NTabs:
    Tab1-订阅: NDataTable(订阅列表 + 操作按钮)
    Tab2-单子: NDataTable(粉丝列表)
    Tab3-审核: 敏感词配置 + 违规记录
    Tab4-备份: 备份列表 + 消息搜索
    Tab5-统计: 图表展示(NGi)

## 8. 客户自助首页

/customer/dashboard 页面:
  NCard: 当前订阅(套餐名, 到期时间, 配额使用)
  NGrid:
    消息统计(今日发送/接收)
    翻译统计(今日字符)
    AI统计(今日对话)
    联系人统计(总联系人)
  最近操作列表

## 9. Pinia Store

| Store | 状态 |
|-------|------|
| useAuthStore | token, user, userType, role, permissions |
| useClientStore | clientList, currentClient, filters |
| useDeptStore | deptTree, currentDept |
| useBillingStore | fundPool, subscriptions, plans |
