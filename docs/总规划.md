## 二、核心设计原则

| 原则       | 说明                                                                               |
| -------- | -------------------------------------------------------------------------------- |
| **本地优先** | 所有消息、联系人、账号配置首先存储在用户电脑的 SQLite 中。云端只做跨网络实时转发，不持久化用户数据。                           |
| **多端实时** | 桌面端与手机端通过 MQTT 实时同步新消息，手机端可通过请求-响应模式拉取历史消息。                                      |
| **隐私安全** | 用户数据不经过任何中心服务器（除 MQTT 短暂转发外），MQTT Broker 不存储消息，仅路由。                              |
| **低成本**  | 公网 MQTT Broker 可使用 EMQX Cloud 免费额度（10 连接/1000 消息/秒），或者自建轻量 Mosquitto（~5$/月 VPS）。 |
| **高防封**  | CloakBrowser 提供源码级指纹伪装，支持独立代理，极大降低平台风控风险。                                        |

## 三、技术选型清单

| 层级                 | 技术                                         | 说明                     |
| ------------------ | ------------------------------------------ | ---------------------- |
| **桌面 UI**          | Tauri 2.0 + Vue 3 + TypeScript + Vite      | 轻量、安全、性能好              |
| **桌面后端**           | Rust (Tauri 自带) + `tokio`                  | 异步运行时，管理子进程和 CDP       |
| **指纹浏览器引擎**        | CloakBrowser + Playwright                  | 源码级 Chromium 补丁，反检测能力强 |
| **本地数据库**          | SQLite (via `tauri-plugin-sql`)            | 存储消息、联系人、账号配置          |
| **MQTT 客户端**       | `rumqttc` (Rust) / `mqtt_client` (Flutter) | 轻量级 MQTT 客户端库          |
| **公网 MQTT Broker** | EMQX Cloud (免费) 或自建 Mosquitto              | 消息路由，不持久化业务数据          |
| **手机端**            | Flutter (跨平台) 或原生                          | 推荐 Flutter 共享代码        |
| **推送通知** (可选)      | FCM + APNs                                 | 手机离线时推送唤醒（非必须）         |
| **翻译代理**           | Nginx + 动态 IP 池 (自建)                       | 低成本、无配额限制              |



## 六、手机端（Flutter）对接

dart

import 'package:http/http.dart' as http;
import 'dart:convert';
Future<String> translateText(String text, String targetLang) async {
  final url = Uri.parse(
    'https://tg.yituoke.org/translate_a/single?client=gtx&dt=t&sl=auto&tl=$targetLang&q=${Uri.encodeComponent(text)}',
  );
  final response = await http.get(url).timeout(Duration(seconds: 10));
  if (response.statusCode == 200) {
    final List<dynamic> data = jsonDecode(response.body);
    return data[0][0][0] as String;
  }
  return '';
}

## 七、数据流（消息同步）

### 7.1 新消息实时同步（桌面 → 手机）

text

WhatsApp 平台 → CloakBrowser 页面 (CDP) → 桌面 Agent 提取消息
                ↓
        桌面 Agent 写入本地 SQLite
                ↓
        桌面 Agent 发布 MQTT 到主题: metachat/user/{userId}/message/new
                ↓
        公网 MQTT Broker → 手机端订阅该主题 → 实时收到消息
                ↓
        手机端存入本地 SQLite 并展示

### 7.2 历史消息拉取（手机 → 桌面）

手机端打开 App → 发送 MQTT 请求（请求-响应模式）→ 桌面端从 SQLite 查询 → 返回消息列表。

### 7.3 发送消息（手机 → 桌面 → 平台）

手机端发送 MQTT 请求 → 桌面端收到 → 通过 CDP 操作 CloakBrowser 页面 → 发送到平台。

## 八、控制台（管理后台）

### 8.1 功能

- 用户注册/登录（JWT）

- 订阅管理（免费版、专业版、企业版）

- 设备授权（限制同时在线设备数）

- MQTT 凭证动态生成（调用 EMQX REST API）

- 功能开关（Feature Flags）：最大账号数、是否允许手机同步、是否允许代理等

### 8.2 与客户端交互 API

| 端点                                 | 说明                 |
| ---------------------------------- | ------------------ |
| `POST /api/client/login`           | 登录，返回 MQTT 凭证和功能开关 |
| `POST /api/client/device/register` | 设备注册/心跳            |
| `GET /api/client/config`           | 拉取最新配置             |

## 九、部署与运维要点

### 9.1 翻译代理服务器

- **VPS**：海外，1C2G 足够，约 $5-10/月

- **域名**：`tg.yituoke.org`，DNS 解析到 VPS IP（Cloudflare 关闭代理，灰色云）

- **证书**：Let's Encrypt 自动续期

- **定时任务**：每天凌晨更新 IP 池

### 9.2 桌面端分发

- Windows：`.exe` 安装包

- macOS：`.app` / `.dmg`（需要代码签名）

- Linux：`.AppImage` / `.deb`

### 9.3 手机端分发

- iOS：TestFlight 或 App Store

- Android：APK 或应用商店

### 9.4 MQTT Broker

- 推荐 **EMQX Cloud** 免费版（10 并发连接，足够初期使用）

- 或自建 Mosquitto（~$5/月 VPS）

## 十、总结

这套方案实现了：

- ✅ **低成本翻译服务**：自建 Nginx 代理 + 社区 IP 池，无需官方 API 配额。

- ✅ **多账号指纹隔离**：CloakBrowser 提供源码级伪装，降低封号风险。

- ✅ **本地数据优先**：用户消息存储在自己电脑，隐私安全。

- ✅ **多端实时同步**：通过 MQTT 实现跨网络（WiFi/4G）的消息推送和历史拉取。

- ✅ **商业化基础**：控制台管理用户订阅、设备授权。
